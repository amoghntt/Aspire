Quality Prediction

Using this Tab, The user can do all the following Predictions for predicting project quality for future releases/modules based on certain prediction questions, referring to the historical data present for the previous releases for the project selected from the data base.

The predictions considered are:

  • A) Defect Density.
  • B) Defect Deferral Rate.
  • C) Defect Acceptance
  • D) All Defects.
  • E) Functional Defects.
  • F) Defective Modules.

Modules as categorized as following based on “RISKS”

  • Normal Modules--> Green
  • Medium Risk Modules--> yellow
  • High Risk Modules-->Red

For each prediction question there are specific metric(s) associated with them. System predicts the values for the metric(s) for future release and finds out the status of the project for a particular Release.

  • 1) Green: NORMAL
  • 2) Amber: MEDIUM RISK
  • 3) Red: HIGH RISK

This prediction helps in taking the necessary actions to control the “Quality” for future releases by controlling the “Defects in the modules”.

Test Optimization

Using this Tab,The user can Classify Test cases and segregate the Test cases according to various categories.

The user is able to Identify duplicate and redundant test cases

  • 1) Duplicate Test Cases--> >90% match
  • 2) Redundant test cases--> 70% - 90% match

Test Coverage

Using this Tab, The user can do all the following actions for calculating the Test coverage (Average, Poor or Good)

The user is able to calculate the “Adequate Test Coverage” by selecting the project and the “Specific Requirement” from the Requirement Specification Document and checks for the Test case coverage. Referring to the historical data present for the previous releases for the project selected from the data base the “Test Case Coverage” is determined.

  • 1) Green: NORMAL--> Test coverage is Good.
  • 2) Amber: MEDIUM RISK--> Test coverage is Average.
  • 3) Red: HIGH RISK-->Test coverage is Poor.
