Installation Guide

Introduction

CRESTA, an abbreviation for Comprehensive Robotic Engine for Software Test Acceleration is a web based application that works on the principle of machine learning and aims at resolving various aspects of Software testing processes by aiding the user in simplifying software testing practices through its various use cases namely Quality Prediction, Test Optimization, Test Coverage and Adaptive Planning, and thereby offering an all the round acceleration to quality assurance and testing of software. Hence CRESTA can serve as not only a test accelerator and test case optimizer but also a predictor of quality of future releases based on the current and past releases.

Purpose

This document is provided to serve as a template for customers regarding installation instructions for CRESTA software application by educating the user through the detailed steps of how to set up and build the entire CRESTA machine and run it successfully.

This procedure is an obligation for contractors to provide installation instructions in order to reduce possible manipulation mistakes, errors, misinterpretations, loss of time to find the correct information, etc., during the software installation phase for the integration team.

Scope

This document is intended for the customers to install CRESTA application seamlessly and without any technical challenges.

Integrators are not experts of the application, hence the need of a step-by-step instructions on how to install.

This document describes how to set up CRESTA mainly based on Docker Image using a Linux server. You can quickly confirm CRESTA working on your typical business PC(Windows) It will take 30 minutes to go through the installation steps, take 30 minutes to run sample kit. Aims to showcase use, not for heavy use. If requesting high performance (such as calculating similarity much faster), you need to design configuration furthermore. This topic is excluded (briefly mentioned in reference section).

User Account Control

This is to be done in windows server application. User Account Control (UAC) is a security component that enables users to perform common tasks as non-administrators, and as administrators without having to switch users, log off or use Run As. User accounts that are members of the local administrators group run most applications as a standard user. By separating user and administrator functions, UAC helps users move towards using standard user rights by default.

Install Python Libraries

User can download python library files from the following URL:

https://pypi.python.org/pypi/

1. Type the following command for Python installation: $ sudo apt-get install python2.7.13

2. Do this for Python 2.7.9 and later (on the python2 series), and Python 3.4 and later include pip by default.

3. To see if pip is installed, open a command prompt and run: $ command –v pip
